# Prompt 模式

把这些模式当成起点使用。生成前要用页面真实文案替换里面的主题细节。

## 通用约束

大多数 prompt 都要加入这些约束：

- 面向用户的资产规划、prompt 摘要、校验说明和最终汇报使用中文。实际图像生成 prompt 可以保留英文，以保证生成质量。
- 明确最终尺寸和方向：`<width>x<height>`。
- 页面功能图 / 用例图要干净、可上线、有生产级质感。
- 使用大品牌功能页构图：一个主视觉、充足留白、高级光影、安静背景。
- 最多 3 个视觉模块：1 个主图 / 主结果，加最多 2 个辅助元素，例如细节放大、前后对比卡或一张次级图卡。
- 主体应占画面约 60-75%。
- 避免复杂拼贴、散落缩略图、道具堆满的桌面、拥挤分镜和大量小 UI 面板。
- 不要出现可读文字、logo、水印、品牌标识、假 UI 文本或复制网页截图。
- 当前页面图片都视为临时占位图，不能当作参考图复用。

## 大品牌风格方向

除非页面文案或用户要求指向其他风格，默认从下面两类风格中选择。

## 同批布局差异规则

当一次生成 6 张功能页配图时，先为整批图片分配不同的布局 archetype。统一高级感可以保留，但展示方式要错开：

- 不要让多张图同时使用相同的“居中大圆角结果卡 + 右后方小副卡 + 浅色影棚产品图”组合。
- 让相邻图片至少改变 3 个维度：主体类型、主体比例、卡片层级、辅助元素、背景/光线、镜头距离、构图方向。
- 可混用这些布局：单张沉浸式结果图、前后对比切片、细节放大、轻量参考卡、少量变体堆叠、场景化成片、抽象编辑轨迹、格式适配卡。
- 语义优先，不为追求差异强行套固定品类模板；每张仍要表达对应文案的核心利益点。
- 生成后用 contact sheet 抽检整体重复度；如果看起来像同一套模板的换皮，重写重复项 prompt。

### 语义优先的 AI 展示风

适用于 AI 模型页、图像生成器、text-to-image 工具、多模态工具和高级创作模型页。

视觉语言：

- 背景不固定深色或浅色；根据页面语义选择高级中性影棚、柔和渐变舞台、明亮产品展示或低饱和深色氛围。
- 大尺寸圆角图片卡，带细描边、柔和阴影和克制发光。
- 舞台感构图，以一张大结果图作为焦点。
- 可选一个圆形细节放大、一个前后对比切片，或一张次级卡片来解释能力。
- 只有在需要时才使用极简抽象 UI 形状；不要让图片像真实产品截图。

可加入英文 prompt 的短句：

```text
Premium AI image-generation showcase, one oversized hero image card as the focal point, background tone chosen to match the feature semantics, subtle rounded border, soft shadow, restrained light accents, generous negative space, maximum two supporting visual elements, no busy collage.
```

### 明亮干净的图像增强展示风

适用于 image enhancer、photo editor、background remover、upscaler、restore、colorize 和 retouching 页面。

视觉语言：

- 大照片优先，背景使用白色、浅灰、柔和蓝色或柔和粉色。
- 使用简单的前后分割、左右对比或圆形细节放大。
- 消费级、亲和、干净；由照片结果本身承担主要视觉吸引力。
- 道具和 UI 元素非常少。
- 自然光、真实纹理，直接展示增强效果。

可加入英文 prompt 的短句：

```text
Bright clean photo-enhancement showcase, one large realistic image result, simple before/after comparison or circular detail zoom, soft studio backdrop, generous whitespace, minimal supporting UI, no busy collage.
```

## 功能：统一生成与编辑

把“文本 / 图片统一生成编辑工作流”表现为一张大结果图，只保留极少辅助上下文。

适合的视觉元素：

- 一张超大的生成结果图。
- 一张小型抽象 prompt / 参考图卡。
- 一道克制的转化光效。
- 一个细窄的前后对比边缘。

避免把图片做成真实产品 UI 截图。避免同一张图里出现很多 prompt 卡、缩略图、设置面板、桌面和道具。

## 功能：速度与快速迭代

表达快速出图和快速试错，但不要使用速度表这类俗套符号。

适合的视觉元素：

- 一张被选中的大结果图。
- 背后叠 1-2 张轻微虚化或半透明的变体卡片。
- 干净的堆叠卡片动势。
- 一个很小的时间线式辅助元素。

最终画面要清晰。动势不应该让图片显得模糊。不要展示几十张缩略图。

## 功能：多图参考 / 一致性

用一个占主导地位的最终主体，加少量参考元素来表现一致性。

适合的视觉元素：

- 一张最终角色 / 产品 / 结果图卡。
- 一张参考图卡。
- 一个细节放大镜，用来展示重复出现的设计细节。

不要出现可读的 "before" 或 "after" 标签。默认避免完整 mood board 拼贴。

## 用例：社交媒体

用一张有冲击力的 campaign visual 作为主视觉，最多在背后加一张次级裁切或格式适配卡。

使用抽象社交 / 帖子框架，不要出现平台标志。避免真实社交媒体 logo、重复缩略图和密集 campaign board。

## 用例：电商

表现一张产品照片被转化为专业销售图片。

适合的视觉元素：

- 单个产品放在干净表面上。
- 轻微前后对比边缘。
- 更专业的光线。
- 一个用于展示质感的圆形细节放大。

除非用户提供了明确允许使用的文案，否则不要出现假的品牌包装文字。除非页面明确强调批量生成，否则不要默认展示多产品网格。

## 用例：概念艺术 / 视觉头脑风暴

用一张强主视觉的概念艺术结果图表现创意探索，而不是展示复杂过程。

适合的视觉元素：

- 一张大的原创角色 / 环境概念图。
- 一个很淡的草图层。
- 一张次级角度卡片。
- 电影感光影。

不要使用受版权保护的角色或可识别的影视 / 游戏风格，除非用户明确要求且政策允许。默认避免拥挤角色设定表、多格分镜和道具堆满的艺术家桌面。
