# Seedance 2.0 视频效果复刻提示词生成器设计

## 背景

当前目标不是做一个单纯给运营人员配置后台的模板说明工具，而是做一个面向终端用户的视频效果复刻工具：用户提供一个参考视频 URL，并上传自己的目标图片，系统根据参考视频逆向出 Seedance 2.0 可用的提示词，让用户能够尽量复刻原视频的运镜、动作节奏、转场、特效和整体视觉效果。

运营人员仍然是重要使用者，但运营侧的主要需求是用中文理解、审核、管理模板和与团队沟通。最终生成给 Seedance 使用的提示词不强制必须是中文，可以是中文、英文或中英混合，优先级是生成效果稳定、表达准确、符合 Seedance 2.0 的素材引用规则。

## 第一版定位

第一版按 **模板生成器** 做，但这里的“模板”不是后台 JSON 配置，而是一套可被终端用户套用的效果复刻逻辑：

- 参考视频 URL 用来生成或保存“原视频效果复刻提示词”。
- 用户上传自己的图片，作为 `@图片1` 或对应目标主体素材。
- 系统把参考视频效果和用户图片组合成最终 Seedance 提示词。
- 用户拿这个提示词去 Seedance 或其他支持 Seedance 2.0 的生成入口使用。
- 第一版暂不接入 KIE，也不在本项目里真实生成视频。

## 目标

第一版要实现：

- 接收参考视频 URL。
- 接收终端用户上传或指定的一张目标图片。
- 基于参考视频生成结构化效果拆解，包括主体替换方式、运镜、动作、节奏、转场、特效、音效和风格。
- 输出最终可复制的 Seedance 2.0 提示词，用于让用户图片尽量复刻参考视频效果。
- 页面用中文和运营人员、终端用户沟通；最终提示词语言不设硬性限制，以效果最优为准。
- 明确提示词中的素材引用用途，例如 `@图片1 作为用户上传的目标主体图`、`@视频1 作为参考视频，用于复刻运镜、动作节奏、转场和视觉特效`。

## 不做的事情

第一版不做以下内容：

- 不接入 KIE `bytedance/seedance-2`。
- 不真实生成视频。
- 不做视频生成任务轮询、失败重试、生成结果存储。
- 不做支付、Premium 权限、账号系统。
- 不要求输出 JSON 给运营人员复制。
- 不自动下载无法直接访问的视频平台资源。
- 不保证每个公开视频 URL 都能被机器准确理解；如果无法直接分析视频，需要允许用户补充文字描述或关键帧信息。

## 产品入口：网页优先，Skill 辅助

第一版推荐用 **网页页面承载终端用户流程**，不建议把 skill 作为主产品入口。

原因：

- 终端用户需要上传图片、查看参考视频、查看效果拆解、复制最终提示词，这些更适合网页交互。
- Higgsfield viral presets 这种产品形态本质是视频模板卡片和上传入口，也更接近 Web UI。
- Skill 更适合内部复用：例如给运营人员或开发者快速输入 URL，生成一份模板草稿；它不适合作为普通用户的唯一入口。

因此第一版架构建议是：

- Web UI：终端用户主入口。
- Prompt 逆向/渲染模块：核心能力，可被网页调用，也可以后续封装成 skill。
- Skill：可选内部工具，不作为第一版主路径。

## 产品流程

1. 用户打开视频效果复刻页面。
2. 用户输入参考视频 URL。
3. 用户上传自己的目标图片。
4. 系统展示参考视频预览；如果无法预览，提示用户补充视频描述或关键画面信息。
5. 系统生成中文效果拆解，帮助用户和运营人员理解复刻逻辑。
6. 系统生成最终 Seedance 2.0 提示词。
7. 用户复制提示词，并将自己的图片和参考视频按提示词中的 `@图片1`、`@视频1` 规则放入 Seedance 生成入口。

后续如果接入真实视频生成接口，第 7 步可以替换为“点击生成视频”。但第一版先只做到提示词复刻。

## 页面输出格式

页面输出应该是中文说明 + 最终提示词，不是 JSON。

推荐展示结构：

```text
参考视频效果：
这个视频的核心效果是主体替换 + 快速推进镜头 + 环绕运镜 + 能量粒子转场。节奏偏短视频爆款，前 3 秒建立主体，中段出现特效爆发，最后定格高光画面。

用户素材：
用户上传的图片作为 @图片1，用来替换参考视频中的主体。参考视频作为 @视频1，用来复刻运镜、动作节奏、转场和视觉特效。

分镜拆解：
0-3秒：镜头从 @图片1 的主体中近景开始，缓慢推近，主体保持清晰，背景出现轻微能量纹理。
3-6秒：参考 @视频1 的环绕镜头和运动节奏，主体周围出现粒子拖影和空间扭曲。
6-10秒：镜头进入高速穿梭或主观视角，完成主要特效爆发，最后定格在主体高光画面。

最终 Seedance 提示词：
@图片1 is the target subject image uploaded by the user. @视频1 is the reference video for camera movement, motion rhythm, transitions, and visual effects. Replace the subject in @视频1 with the subject from @图片1, while preserving the fast push-in camera, orbiting movement, particle energy transition, spatial distortion, and final hero freeze-frame...
```

说明部分优先中文；最终提示词可以中文、英文或中英混合。只要能更准确驱动 Seedance 2.0 复刻效果，就允许使用英文描述。

## Seedance 提示词规则

提示词生成必须遵守 Seedance 2.0 的核心规则：

- 每个素材引用都必须说明用途。
- `@图片1` 默认代表用户上传的目标主体图，除非用户明确选择它作为首帧、背景或产品图。
- `@视频1` 默认代表参考视频，用于复刻运镜、动作节奏、转场、特效和音效。
- 提示词必须描述主体、场景、动作、运镜、分时段、转场/特效、音频/音效、风格氛围。
- 8-10 秒以上的视频建议使用分镜时间轴。
- 爆款模板要优先复刻参考视频的节奏和视觉机制，而不是只描述静态画面风格。
- 如果用户上传写实真人脸部素材，需要提示可能存在平台拦截或效果不稳定。

## 逆向分析方式

第一版可以先支持两种输入情况：

1. 可直接预览或访问的视频 URL：
   - 页面展示参考视频。
   - 系统尝试基于 URL 和用户补充信息生成效果拆解。

2. 无法直接分析的视频 URL：
   - 页面提示用户补充视频描述、关键帧描述或人工观察到的特效信息。
   - 系统根据用户补充内容生成提示词。

如果未来运行环境支持视频理解或抽帧分析，可以加入自动抽帧、关键帧识别、镜头节奏分析等能力。但第一版不能把“必须自动理解任意 URL 视频”作为完成条件。

## 信息架构

第一版建议新增两个核心页面：

- `/video-effects`：视频效果复刻首页，包含参考视频 URL 输入、图片上传、生成按钮和最近模板列表。
- `/video-effects/[id]`：结果页，展示参考视频、用户图片、中文效果拆解、分镜时间轴和最终提示词。

如果后续需要做类似 Higgsfield 的模板库，再扩展：

- `/video-presets`：展示已经沉淀的热门效果模板。
- `/video-presets/[id]`：用户选择已有模板，上传自己的图片，直接生成最终提示词。

## 数据存储

第一版仍可用本地文件存储，保存用户生成过的效果复刻记录。

建议内部保存字段：

- `id`
- `sourceVideoUrl`
- `targetImagePath`
- `targetImageRole`
- `effectSummary`
- `timeline`
- `finalPrompt`
- `promptLanguage`
- `warnings`
- `createdAt`
- `updatedAt`

这些字段只是内部实现细节。页面展示时必须转成中文说明和最终提示词，不直接展示 JSON。

## 错误处理

需要处理：

- 参考视频 URL 为空或格式非法。
- 用户未上传目标图片。
- 参考视频无法预览。
- 参考视频无法被自动分析，需要用户补充描述。
- 生成的提示词没有明确 `@图片1` 或 `@视频1` 的用途。
- 提示词缺少运镜、动作节奏、转场、特效等核心复刻信息。
- 用户素材可能涉及写实真人脸部时，展示风险提醒。

## 测试要求

测试应覆盖：

- 没有参考视频 URL 时不能生成。
- 没有目标图片时不能生成最终复刻提示词。
- 最终提示词必须包含 `@图片1` 和 `@视频1` 的用途说明。
- 最终提示词必须包含运镜、动作节奏、转场/特效、分时段或等价结构。
- 页面输出不以 JSON 作为主展示格式。
- KIE 不在第一版流程中被调用。
- 存储模块能保存、读取、列出效果复刻记录。

## 后续扩展

后续可以分三条线扩展：

- 模板库：把常用参考视频效果沉淀成类似 Higgsfield viral presets 的卡片库。
- Skill：把提示词逆向和渲染能力封装成内部 skill，方便运营或开发者在非网页环境调用。
- 视频生成接口：等提示词质量稳定后，再单独设计 KIE 或其他视频生成 provider 的接入，不和第一版提示词复刻工具混在一起。
