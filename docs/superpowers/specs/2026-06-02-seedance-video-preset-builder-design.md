# Seedance 2.0 视频效果逆向 Skill 设计

## 背景

当前第一版不是给终端用户直接使用的网页产品，而是给内部人员使用的提示词逆向工具。典型使用方式是：内部人员只提供一个参考视频 URL，系统尽量根据视频内容逆向出 Seedance 2.0 可用的效果复刻提示词模板。这个模板后续可以被运营人员放到其他项目后台，或者沉淀成类似 Higgsfield viral presets 的视频特效模板。

因此第一版入口应优先做成 **skill**，而不是网页页面。网页更适合后续面向终端用户的上传图片、选择模板、生成视频流程；但当前内部使用者希望减少操作，只把视频 URL 丢给 Codex，让 Codex 直接输出模板内容。

运营人员和内部团队用中文沟通，所以分析说明、模板解释、使用备注应使用中文。最终给 Seedance 的提示词不强制必须中文，可以中文、英文或中英混合，优先选择更容易稳定复刻效果的表达。

## 第一版定位

第一版做一个内部 skill：`seedance-video-preset-reverse`。

它的职责是：

- 接收一个参考视频 URL。
- 尽量理解视频里的运镜、主体动作、节奏、转场、特效、音效和视觉风格。
- 输出一个可复用的 Seedance 2.0 效果复刻提示词模板。
- 默认模板里预留 `@图片1` 作为未来用户上传的目标主体图，`@视频1` 作为参考视频。
- 输出中文分析说明，方便运营人员审核和配置。
- 不接入 KIE，不真实生成视频。

## 目标

第一版要实现：

- 内部人员只输入视频 URL 时，skill 能尽量产出完整模板。
- 输出内容包含模板名称、适用素材、效果概述、分镜时间轴、最终提示词、使用备注。
- 提示词必须明确 `@图片1` 和 `@视频1` 的用途。
- 提示词要能让未来用户上传自己的图片后，尽量复刻参考视频效果。
- 如果无法直接理解视频内容，skill 才进行最小化追问，例如要求补充一句视频描述、关键画面、或上传截图。
- 输出格式不是 JSON，而是可直接阅读和复制的中文结构化文本。

## 不做的事情

第一版不做以下内容：

- 不做网页页面。
- 不接入 KIE `bytedance/seedance-2`。
- 不真实生成视频。
- 不做终端用户上传图片流程。
- 不做模板库卡片页面。
- 不做支付、Premium 权限、账号系统。
- 不把 JSON 作为主输出。
- 不保证任意视频平台 URL 都能被自动读取；无法读取时走最小化追问。

## 使用方式

内部人员直接对 Codex 说：

```text
用 seedance 逆向这个视频：https://example.com/video.mp4
```

或者：

```text
这个视频做成 Seedance 模板：https://example.com/demo
```

skill 被唤起后：

1. 识别参考视频 URL。
2. 判断该 URL 是否可直接访问、预览或分析。
3. 如果能分析，直接输出模板。
4. 如果不能分析，最多先问一个问题，要求用户补充视频描述或关键帧。
5. 生成最终结果。

## 输出格式

输出必须是中文结构化文本，面向内部运营人员阅读和复制。

推荐格式：

```text
模板名称：
未来感眼镜穿梭特效

适用素材：
@图片1：用户未来上传的目标主体图，用来替换参考视频中的主体。
@视频1：当前参考视频，用来复刻运镜、动作节奏、转场、视觉特效和音效节奏。

效果概述：
这个视频的核心效果是主体替换 + 快速推进镜头 + 环绕运镜 + 能量粒子转场。整体节奏偏短视频爆款，前段建立主体，中段出现空间扭曲和粒子爆发，最后定格在主体高光画面。

分镜时间轴：
0-3秒：镜头从 @图片1 的主体中近景开始，缓慢推近，主体保持清晰，背景出现轻微能量纹理。
3-6秒：参考 @视频1 的环绕镜头和运动节奏，主体周围出现粒子拖影、空间扭曲和发光边缘。
6-10秒：镜头进入高速穿梭或主观视角，完成主要特效爆发，最后定格在主体高光画面。

最终 Seedance 提示词：
@图片1 作为目标主体图，@视频1 作为参考视频。将 @视频1 的运镜、动作节奏、转场方式、视觉特效和音效节奏复刻到 @图片1 的主体上。保持参考视频中的快速推进镜头、环绕运动、粒子拖影、空间扭曲和最终高光定格...

使用备注：
建议生成 8-10 秒，竖屏 9:16。若目标素材是写实真人脸部，可能存在平台拦截或一致性不稳定。
```

最终提示词可以使用中文、英文或中英混合。示例里用中文是为了方便内部审核；实际生成时，如果英文表达更清楚，可以输出英文提示词。

## Seedance 提示词规则

skill 生成提示词时必须遵守 Seedance 2.0 的核心规则：

- 每个素材引用都必须说明用途。
- `@图片1` 默认代表未来用户上传的目标主体图。
- `@视频1` 默认代表参考视频，用于复刻运镜、动作节奏、转场、特效和音效。
- 提示词必须覆盖主体、场景、动作、运镜、分时段、转场/特效、音频/音效、风格氛围。
- 8-10 秒以上的视频建议使用分镜时间轴。
- 爆款模板要优先复刻参考视频的节奏和视觉机制，而不是只描述静态画面风格。
- 如果模板依赖写实真人脸部，需要在使用备注里提示风险。

## URL 分析策略

skill 以“少问问题”为原则：

1. 如果 URL 是直接视频文件、可公开访问页面、或当前工具可以读取的网页：
   - 尽量直接分析视频或页面上下文。
   - 输出模板，不要求用户额外补充。

2. 如果 URL 无法访问、无法播放、或无法判断视频内容：
   - 只问一个最小问题，例如：“这个视频的核心画面和特效是什么？可以用一句话描述，或发 1-3 张关键帧截图。”
   - 用户补充后继续输出模板。

3. 如果用户明确说“不想补充”，skill 仍应基于 URL、标题、页面可见信息和常见视频模板规律生成一个低置信度版本，并标注“不确定项”。

## 信息架构

第一版不需要新增网页页面。

建议实现为本地 skill：

- skill 名称：`seedance-video-preset-reverse`
- skill 输入：参考视频 URL，可选补充描述，可选关键帧截图。
- skill 输出：中文结构化模板 + 最终 Seedance 提示词。
- skill 依赖：Seedance 2.0 提示词指南、视频 URL 读取/浏览能力、必要时的图片或截图理解能力。

后续再考虑：

- Web 页面：给终端用户上传图片、选择模板、复制提示词或生成视频。
- 模板库：沉淀常用视频效果模板，做成类似 Higgsfield viral presets 的卡片库。
- KIE 接入：等提示词质量稳定后，再单独设计视频生成 provider。

## 存储策略

第一版 skill 可以先不做持久化存储。每次根据 URL 输出一份模板即可。

如果需要沉淀模板，再保存以下内部字段：

- `id`
- `sourceVideoUrl`
- `title`
- `effectSummary`
- `timeline`
- `assetInstructions`
- `finalPrompt`
- `notes`
- `confidence`
- `createdAt`

这些字段只是后续模板库的内部数据，不作为第一版主输出。

## 错误处理

需要处理：

- 没有视频 URL。
- URL 无法访问。
- URL 不是视频或无法判断视频内容。
- 用户不愿补充视频描述。
- 生成的提示词没有明确 `@图片1` 或 `@视频1` 的用途。
- 提示词缺少运镜、动作节奏、转场、特效等核心复刻信息。

## 测试要求

测试或人工验收应覆盖：

- 只给一个视频 URL 时，skill 输出完整模板结构。
- 输出包含模板名称、适用素材、效果概述、分镜时间轴、最终提示词、使用备注。
- 最终提示词包含 `@图片1` 和 `@视频1` 的用途说明。
- 最终提示词包含运镜、动作节奏、转场/特效、分时段或等价结构。
- URL 无法分析时，skill 只提出最小必要追问。
- KIE 不在第一版流程中被调用。

## 后续扩展

后续扩展顺序建议：

1. 先做 skill，把“只给 URL 就能产出提示词模板”的内部工作流跑通。
2. 再沉淀模板库，把常用 URL 生成的模板保存为可复用 preset。
3. 再做网页，让终端用户上传目标图片、选择模板、复制提示词。
4. 最后再考虑接入 KIE 或其他视频生成接口，实现完整生成闭环。
